Redis

初识Redis

数据库主要涉及到两个方面：数据的write与read
NoSQL的数据库：HBase/Redis/MangoDB…
Redis：是一个高性能的Key-Value的数据库
大负荷、大负载的应用，不管如何优化，在RDBMS中很难去达到预想的性能指标，故自己开发了一个基于内存的数据库-》Redis
在N多条记录中根据条件去快速查找某几条记录-》Redis中很快

  • 面向Key-Value对的数据类型的内存数据库，满足读写需求
  • Key通常是String
  • Value的类型较多，字符串String、哈希表Hash、字符串列表list、字符串集合set、有序集合sorted set…

    Redis特性

  • 速度快：10w QPS。主要原因是数据大部分在内存中（对比HDFS的NN）,一个Redis是一个单线程，在内存中访问速度是最快的
  • 持久化：可以将内存中的数据每隔一段时间就保存到磁盘上，重启的时候可以从磁盘上加载到内存。RDB：周期性的快照的方式，AOF：日志的方式，所有写操作的命令保存到日志里面（很像HDFS的NN和2NN），这两种方式可以同时使用，重启的时候优先AOF，因为AOF的数据通常比RDB更完整
  • 多种数据结构：strings, hashes, lists, sets, sorted sets with range queries, bitmaps等等
  • 多种编程语言：Java、PHP、Python、Ruby、Lua、NodeJS等等
  • 丰富的功能：发布订阅、Lua脚本、事务、Pipeline（管道，把多个指令放进去一次性执行）
  • 简单易用：不依赖外部库、单线程
  • 主从复制：主节点的数据会有从节点的数据作为副本，保证了HA
  • 高可用和分布式：Redis-Sentinel(2.8)和Redis-Cluster(3.0)

    应用场景

  • Cache缓存系统：从只能存数据变成能够更新数据

多数据库特性

支持多数据库，Redis启动时自动帮我们创建，以0为编号递增的，默认支持16个数据库，可通过redis.conf修改
redis-cli默认选择0号数据库，可以通过select dbid切换
多数据库之间一般是相互隔离的，但是也有特例
可以在某一数据库中删除所有数据库的内容
