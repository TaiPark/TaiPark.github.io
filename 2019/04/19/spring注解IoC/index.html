注解

注解配置与xml配置要实现的功能都是一样的。

  • 用于创建对象的注解
    与在xml配置文件中编写一个\<bean>标签实现的功能一样
    @Component：用于把当前类对象存入spring容器中

    属性：

    value:用于指定bean的id，当不写时，它的默认值是当前类名且首字母小写

    bean.xml：

    以下三个注解与@Component一样，他们三个是spring框架为我们提供明确三层使用的注解，使我们三层对象更加清晰：
    @Controller：表现层
    @Service：业务层
    @Repository：持久层

  • 用于注入数据的注解
    与在xml配置文件中编写一个\<bean>标签中写一个\<property>标签实现的功能一样
    @Autowired：自动按照类型注入，只要容器中有唯一的一个bean对象类型和要注入的变量类型匹配就能够注入成功
    书写位置可以是成员变量上，也可以是方法上。

    Spring的IoC容器：Map结构
    |key(String)|value(Object)|
    |:-:|:-:|
    |accountService|public class AccountServiceImpl implements IAccountService|
    |accountDao|public class AccountServiceImpl implements IAccountDao|

    @Autowired寻找value(Object),当容器中数据类型有一致的时自动按照类型注入
    |private|IAccountDao|accountDao| = |null|
    |:-:|:-:|:-:|:-:|:-:|
    |访问修饰符|数据类型|变量名称||数据类型|
    如果IoC容器中有多个容器匹配时，先按类型圈定出匹配的对象，再使用变量名称作为ID在圈定范围内继续查找，如果有一样的也可以注入成功，若都不一样则报错
    @Qualifier：在按照类中注入的基础之上再按照名称注入。它在给类成员注入时不能单独使用（不然会报空指针异常，要和@Autowired一起用）。但是在给方法参数注入时可以
    属性：value：用于指定注入bean的ID

    @Resource：直接按照bean的id注入。它可以独立使用
    属性：name:用于指定注入bean的ID

    以上三个注入都只能注入其他bean类型的数据，而基本类型和String类型无法使用上述注解实现。另外，集合类型的注入只能通过XML来实现。

    @Value：用于注入基本类型和String类型的数据
    属性：value:用于指定数据的值，可以使用spring中的SpEL(spring的el表达式)
    SpEL：${表达式}

  • 用于改变作用范围的注解
    与在xml配置文件中编写一个\<bean>标签写一个\<scope>实现的功能一样
    Scope：用于指定bean的作用范围
    属性：value：指定范围的取值。常用取值：singleton prototype

  • 和生命周期相关的
    与在xml配置文件中编写一个\<bean>标签写一个\<init-method>和\<destroy-methode>实现的功能一样
    @PreDestroy：用于指定销毁方法
    @PostConstruct：用于指定初始化方法

    spring不负责多例对象的销毁
